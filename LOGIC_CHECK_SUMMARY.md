## 发现并修复的问题

### 🔴 严重问题（已修复）

**问题**: NO 订单的第一档检查使用了错误的价格基准

**原因**: 使用 YES 的第一档价格来检查 NO 订单，导致误判

**修复**:
```typescript
// 修复前（错误）
if (lastPrices.noBid > 0 && lastPrices.noBid >= bestBid * 0.999) {
  // bestBid 是 YES 的价格，不应该用于检查 NO
}

// 修复后（正确）
const noBestBid = 1 - yesBestAsk;  // NO 的买价 = 1 - YES 的卖价
const noBestAsk = 1 - yesBestBid;  // NO 的卖价 = 1 - YES 的买价

if (lastPrices.noBid > 0 && lastPrices.noBid >= noBestBid * 0.999) {
  // 使用 NO 的第一档价格检查 ✅
}
```

**影响**: 防止了 NO 订单被误判为第一档，避免了不必要的重新挂单
