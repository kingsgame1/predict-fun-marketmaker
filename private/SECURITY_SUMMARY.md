## 🛡️ 防破解机制

### 1. 代码分离

| 攻击方式 | 防护措施 |
|---------|---------|
| 从 GitHub 获取生成工具 | ❌ 生成工具不在公开代码库 |
| 查看 SECRET_KEY | ❌ 密钥不在公开代码库 |
| 修改代码绕过验证 | ⚠️ 需要修改 TypeScript 并重新编译 |

### 2. 激活码加密

```typescript
// 激活码 = SHA256(用户数据 + 机器ID + SECRET_KEY)
const encrypted = sha256(userId + userName + machineId + expireDate + SECRET_KEY);
// 格式: XXXX-XXXX-XXXX-XXXX-XXXX-XXXX-XXXX-XXXX
```

**特性**:
- ✅ 单向哈希，不可逆
- ✅ 无法从激活码推导出 SECRET_KEY
- ✅ 无法从激活码推导出生成参数

### 3. 机器绑定

```typescript
// 机器ID = SHA256(操作系统 + CPU架构 + MAC地址)
const machineId = sha256(platform + arch + mac);
```

**防护**:
- ✅ 激活码只能在特定机器上使用
- ✅ 无法通过复制激活文件共享
- ✅ 虚拟机克隆也需要新激活码

### 4. 时间限制

```typescript
// 激活码包含过期时间戳
const expireDate = Date.now() + days * 24 * 60 * 60 * 1000;

// 每次启动验证过期时间
if (activation.expireDate < Date.now()) {
  throw new Error('激活码已过期');
}
```

**防护**:
- ✅ 过期自动失效
- ✅ 可设置任意有效期
- ✅ 需要续费才能继续使用

### 5. 编译后代码

**生产环境部署**:
```bash
# 编译 TypeScript
npm run build

# 分发编译后的 dist/ 目录
# 用户拿不到源代码
```

**增加破解难度**:
- ⚠️ TypeScript 编译为 JavaScript 后可读性降低
- ⚠️ 可以进一步使用 JavaScript Obfuscator 混淆
- ⚠️ 关键逻辑可以拆分和加密

## 🚨 可能的攻击及防护

### 攻击1: 修改激活检查代码

**方法**: 用户修改 `src/arbitrage-bot.ts` 删除激活检查

**难度**: ⭐⭐⭐ (中等)

**防护**:
1. 分发编译后的代码，不分发源代码
2. 使用代码混淆器增加难度
3. 关键逻辑分散在多个文件
4. 未来可添加代码签名验证

**当前状态**: ⚠️ 源代码公开，技术用户可以绕过

**建议**: 生产环境分发编译后的 `dist/` 目录

### 攻击2: 系统时间回溯

**方法**: 修改系统时间绕过过期检查

**难度**: ⭐ (简单)

**防护**:
1. 记录上次激活时间戳
2. 检测时间倒流
3. 可选：在线时间验证

**当前状态**: ⚠️ 未实施时间回溯检测

**建议**: 可选增强功能

### 攻击3: 逆向工程生成工具

**方法**: 分析公开代码，重写激活码生成器

**难度**: ⭐⭐⭐⭐⭐ (极难)

**防护**:
1. SECRET_KEY 不在公开代码中
2. SHA-256 单向哈希不可逆
3. 即使知道算法，没有 SECRET_KEY 也无法生成有效激活码

**当前状态**: ✅ 安全

**建议**: 定期更换 SECRET_KEY（影响已有用户）

### 攻击4: 共享激活文件

**方法**: 复制 `.arbitrage_activation.json` 到其他机器

**难度**: ⭐⭐⭐⭐ (很难)

**防护**:
1. 机器绑定检查
2. 验证当前机器 ID 与激活文件中的机器 ID
3. 不匹配则拒绝

**当前状态**: ✅ 安全

**建议**: 已实施
