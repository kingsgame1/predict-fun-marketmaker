# 市场推荐模块 V2 改进说明

## 概述

深度思考 Predict.fun 积分规则，完善市场推荐算法。核心改进是引入"积分资金效率"概念，让推荐更符合实际做市战略。

## 原有问题

1. **只关注"能不能挂"，不关注"挂得好不好"**
   - 同样的 buffer=3c，min_shares=50 和 min_shares=500 的市场得到相同评分
   - 实际上前者资金门槛低10倍，资金效率高得多

2. **没有资金效率概念**
   - 没有考虑到 min_shares 对资金占用的影响
   - 没有衡量单位资金能产生多少 buffer

3. **盘口质量评估不够**
   - 只看价差大小，没看价差占用比例
   - 不能区分"盘口宽松"和"盘口拥挤"

## V2 改进内容

### 1. 积分资金效率评分 (核心)

**公式**: `pointsEfficiency = (bufferPerSide / min_shares) * 1000`

意义：每投入1000股股票，能获得多少缓冲空间（cents）

**Scoring**:
```javascript
score += Math.min(20, Math.floor(pointsEfficiency * 2));
```

例子：
- 市场A: buffer=3c, min_shares=50 → efficiency = 60 → +12分
- 市场B: buffer=3c, min_shares=500 → efficiency = 6 → +1分

### 2. min_shares 门槛评分

| min_shares | 评分 | 说明 |
|-----------|------|------|
| <= 50 | +15 | 低门槛，资金效率高 |
| <= 100 | +10 | 门槛低，适合小资金 |
| <= 200 | +5 | 门槛中等 |
| > 200 | -5 | 门槛高，需要大量资金 |

### 3. Buffer Utilization (盘口拥挤度)

`盘口利用率 = bookSpread / maxSpread`

| 利用率 | 评分 | 说明 |
|--------|------|------|
| < 30% | +10 | 盘口宽松，容易挂单 |
| 30-70% | 0 | 正常 |
| > 70% | -5 | 盘口拥挤，难以挂单 |

### 4. max_spread 奖励

| max_spread | 评分 | 说明 |
|-----------|------|------|
| >= 8c | +8 | 宽价差，盈利空间大 |
| >= 6c | +5 | 标准价差 |
| < 6c | 0 | 严格价差 |

### 5. 流动性评分 (细化)

| 24h Volume | 评分 | 说明 |
|-----------|------|------|
| > $50K | +15 | 极高流动性 |
| > $20K | +12 | 高流动性 |
| > $10K | +8 | 中高流动性 |
| > $2K | +4 | 中等流动性 |
| > $500 | +1 | 低流动性 |
| <= $500 | -10 | 极低流动性，警告 |

### 6. 时间风险评分 (细化)

| 到期时间 | 评分 | 说明 |
|---------|------|------|
| > 30天 | +8 | 长期市场，稳定 |
| 14-30天 | +10 | 适中期限（最优） |
| 7-14天 | +5 | 2周内到期 |
| 3-7天 | +2 | 1周内到期⚠️ |
| 1-3天 | -10 | 3天内到期❗ |
| < 1天 | -30 | 24h内到期🚫 |

### 7. 价格质量

| 中价价格 | 评分 | 说明 |
|---------|------|------|
| 0.1-0.9 | +3 | 正常价格区间 |
| < 0.05 或 > 0.95 | -5 | 极端价格，风险高 |

### 8. Polymarket 日率评分

| daily_rate | 评分 | 说明 |
|-----------|------|------|
| > $0.05 | +8 | 高返利市场 |
| > $0.01 | +4 | 中等返利 |
| > 0 | +2 | 有返利 |

## UI 改进

### 市场卡片显示

1. **积分规则信息**：显示 `max_spread_cents` 和 `min_shares`
   - 例如："6c/100股" 表示 max_spread=6美分，min_shares=100股

2. **积分资金效率标签**：显示 `pointsEfficiency`值
   - 高效率（>=0.5）：普通标签
   - 低效率（<0.5）：警告标签

3. **更多评分原因**：
   - 门槛高低
   - 盘口宽松/拥挤
   - 极高/高/中高流动性
   - 长期市场
   - 宽价差
   - 高返利市场

## 使用建议

### 保守模式
- 推荐阈值：score >= 70
- 选择高效率（efficiency >= 1.0）、低门槛（min_shares <= 100）的市场
- 避开即将到期（<7天）的市场

### 激进模式
- 推荐阈值：score >= 45
- 可以接受中等效率、中等门槛的市场
- 适当配置一些高返利的 Polymarket 市场
