# 🔄 双版本客户端使用指南

## 📖 概述

Predict.fun 做市商客户端提供两个版本，满足不同用户的需求：

- **简化版（🎯）**：专为小白用户设计，专注积分获取
- **完整版（🔧）**：面向高级用户，包含所有功能

两个版本可以随时切换，无需重启应用。

## 🔄 版本切换

### 方法1：界面内切换（推荐）⭐⭐⭐⭐⭐

**无需重启，一键切换！**

#### 从完整版切换到简化版
1. 点击右上角 **"🎯 简化版"** 按钮
2. 确认切换
3. ✅ 立即切换到简化版

#### 从简化版切换到完整版
1. 点击右上角 **"🔧 完整版"** 按钮
2. 确认切换
3. ✅ 立即切换到完整版

### 方法2：重置版本选择

点击右上角的 **"🔄 重置"** 按钮：
- 清除版本选择记忆
- 下次启动显示版本选择页面

### 方法3：命令行切换

```bash
cd desktop-app/renderer

# 切换到简化版
mv index.html index_full.html
mv index_simple.html index.html

# 切换回完整版
mv index.html index_simple.html
mv index_full.html index.html

# 重启应用
npm start
```

## ⚠️ 注意事项

### ✅ 切换时保持不变

- **做市商运行状态** - 继续运行，不受影响
- **环境变量配置** - 共享同一个 `.env` 文件
- **日志记录** - 所有日志保持一致
- **WebSocket 连接** - 实时数据不中断

### ⚠️ 需要注意

1. **未保存的修改**
   - 切换前记得保存配置
   - 未保存的修改会丢失

2. **套利机器人**
   - 在简化版中看不到套利控制
   - 但如果正在运行，会继续运行
   - 切换到完整版后可以看到

3. **界面刷新**
   - 切换相当于重新加载页面
   - 所有状态保持不变

## 🎯 积分优化配置

简化版的"一键最佳实践"会自动配置：

```bash
# 核心积分优化
MM_POINTS_PRIORITIZE=true              # 优先积分市场
MM_POINTS_OPTIMIZATION=true             # 启用积分优化
MM_POINTS_V2_OPTIMIZER=true             # V2 极致优化
MM_ONLY_POINTS_MARKETS=true             # 只做积分市场

# 积分规则
MM_POINTS_MIN_SHARES=100                # 最小订单
MM_POINTS_MAX_SPREAD_CENTS=6            # 最大价差
MM_POINTS_ASSUME_ACTIVE=true            # 启用默认规则

# 其他优化
ENABLE_TRADING=true                     # 启用交易
PREDICT_WS_ENABLED=true                  # WebSocket
AUTO_CONFIRM=true                       # 自动确认
```

完整版可以手动配置以上参数，或使用模板按钮。

## 🔧 常见问题

### Q: 如何清除版本选择记忆？

**A**: 点击右上角的 **"🔄 重置"** 按钮，下次启动会显示版本选择页面。

### Q: 两个版本的配置是共享的吗？

**A**: 是的！两个版本使用同一个 `.env` 文件，配置完全共享。

### Q: 切换版本会中断做市商吗？

**A**: 不会！做市商会继续运行，配置和状态都保持不变。

### Q: 我可以从简化版切换到完整版使用套利功能吗？

**A**: 可以！点击右上角的"🔧 完整版"按钮即可切换。

### Q: 简化版的积分优化功能是否和完整版一样？

**A**: 是的！积分优化引擎（V1 和 V2）在两个版本中完全可用。

### Q: 如何知道当前是哪个版本？

**A**:
- 简化版标题："积分获取专用"
- 完整版标题："做市与套利操作台"
- 右上角有切换按钮显示另一个版本
