### 🎮 使用方法

#### 1. 启动应用

```bash
cd desktop-app
npm start
```

#### 2. 激活套利模块

在套利机器人区域：
1. 输入激活码（格式：XXXX-XXXX-XXXX-XXXX-XXXX）
2. 点击"激活"按钮
3. 等待显示"✅ 已激活"

#### 3. 选择套利模式

点击"⚡ 超级高频"按钮，会显示策略配置面板

#### 4. 配置策略

默认推荐配置（前3个策略已启用）：
- ✅ 高概率债券
- ✅ 均值回归
- ✅ 领域专业化

你可以根据需要启用其他策略。

#### 5. 启动机器人

点击"启动套利"按钮，系统会：
1. 检查激活码
2. 显示启用的策略
3. 启动超级高频套利机器人
4. 在运行日志中显示实时输出

### 📝 配置文件

超级高频套利使用配置文件：`config/high-frequency-arb.yml`

```yaml
strategies:
  high_probability_bond: true    # 高概率债券
  mean_reversion: true           # 均值回归
  domain_specialization: true    # 领域专业化
  multi_result: false             # 多结果套利
  information_arbitrage: false   # 信息套利
  cross_platform: false          # 跨平台套利
  yes_no_under: false             # Yes+No<1套利

execution:
  auto_execute: false            # 自动执行
  max_daily_trades: 30           # 最大每日交易数
  scan_interval_ms: 1000         # 扫描间隔

risk_management:
  max_drawdown: 0.15             # 最大回撤15%
  stop_loss: 0.10                # 止损10%
```

### ⚠️ 重要提示

1. **激活码要求**
   - 套利模块需要激活码
   - 激活码有有效期
   - 可以在`.arb-activation.json`中查看激活状态

2. **风险管理**
   - 建议先用模拟模式测试
   - 设置合理的止损和最大回撤
   - 不要启用所有策略

3. **性能优化**
   - 推荐启用前3个策略（标记⭐）
   - 避免启用Yes+No<1套利（竞争激烈）
   - 跨平台套利需要配置多平台API

4. **监控运行**
   - 定期查看运行日志
   - 关注盈亏情况
   - 及时调整策略参数

### 🐛 故障排除

#### 问题1：启动失败

**错误信息：** "当前版本不支持超级高频套利"

**解决方法：**
使用命令行启动：`npm run start:super-hf`

#### 问题2：策略配置未生效

**解决方法：**
1. 检查`config/high-frequency-arb.yml`文件
2. 确保策略设置正确
3. 重启机器人

#### 问题3：激活码无效

**解决方法：**
1. 检查激活码格式：XXXX-XXXX-XXXX-XXXX-XXXX
2. 确认激活码未过期
3. 联系管理员重新生成
