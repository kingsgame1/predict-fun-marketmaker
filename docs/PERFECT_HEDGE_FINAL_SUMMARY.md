### 正确的策略流程

```
1. 初始状态：0 YES + 0 NO（空仓）
   ↓
2. 双边挂单赚积分：
   - YES 卖单（价格较高）
   - NO 卖单（价格较高）
   ↓
3a. 场景A：YES 卖单被吃
    - 被迫买入 10 YES
    - 立即市价买入 10 NO 对冲
    - 结果：10 YES + 10 NO ✅

3b. 场景B：NO 卖单被吃
    - 被迫买入 10 NO
    - 立即市价买入 10 YES 对冲
    - 结果：10 YES + 10 NO ✅
```

## 🎯 策略优势

### 1. 初始无需资金

```
之前（错误）:
  - 需要先买入 100 YES + 100 NO
  - 初始成本：$100

现在（正确）:
  - 初始 0 YES + 0 NO
  - 初始成本：$0 ✅
```

### 2. 主要收益是积分

```
收益来源：
  1. 积分收益（主要）
     - 挂单就能赚取积分
     - 不需要成交

  2. 价差收益（次要）
     - 被吃单后对冲
     - 赚取 bid-ask spread
```

### 3. 风险可控

```
对冲前：10 YES + 0 NO
  - 总价值 = 10 × $0.60 = $6
  - 如果 YES→$0.80：价值 = $8 ❌ 风险

对冲后：10 YES + 10 NO
  - 总价值 = 10 × $0.60 + 10 × $0.40 = $10
  - 如果 YES→$0.80, NO→$0.20：价值 = $8 + $2 = $10 ✅ 无风险
```

### 4. 自动化程度高

```
系统自动：
  ✅ 检测被吃单
  ✅ 计算需要对冲的数量
  ✅ 市价买入对边
  ✅ 验证对冲效果
  ✅ 恢复 1:1 平衡
```

## ⚠️ 关键要点

### 必须满足的条件

1. ✅ **YES + NO ≤ 1.05**
   - 超过这个值对冲失效
   - 实时监控，超标立即平仓

2. ✅ **高流动性**
   - 24小时交易量 > $10,000
   - 订单簿深度 > $1,000
   - 确保能快速成交对冲

3. ✅ **立即对冲**
   - 被吃单后不能延迟
   - 市价单（不限制价）
   - 滑点容忍度 2-3%

### 常见错误

❌ **错误 1**: 在 YES+NO>1.05 的市场使用
- 后果：对冲失效，可能亏损 10-20%

❌ **错误 2**: 挂单价太低
- 后果：频繁被吃，对冲成本高

❌ **错误 3**: 对冲延迟
- 后果：暴露在方向性风险中

❌ **错误 4**: 忽略 YES+NO 变化
- 后果：可能从 1.00 涨到 1.20，对冲失效
