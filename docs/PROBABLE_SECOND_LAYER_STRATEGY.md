# 🎯 Probable 积分优化 - 第二档挂单策略 v2.1

## 策略概述

**核心思想：根据订单簿第一档价格动态调整，挂在第二档，既安全又能最大化得分**

- ❌ **第一档**：太激进，容易被吃单，不安全
- ✅ **第二档**：最佳位置，安全且得分高
- 🔄 **动态调整**：实时读取订单簿，基于第一档价格计算第二档位置

## 工作原理

### 1. 实时读取订单簿第一档价格

系统实时获取订单簿的：
- `bestBid`：第一档买价（例如 99.1）
- `bestAsk`：第一档卖价（例如 99.2）

### 2. 动态计算挂单位置

**新版策略（基于订单簿动态调整）：**

```
我们的买价 = bestBid - MM_TOUCH_BUFFER_FIXED_CENTS
我们的卖价 = bestAsk + MM_TOUCH_BUFFER_FIXED_CENTS
```

**示例：**
```
订单簿状态：
买一：99.1（100股）
卖一：99.2（80股）

我们的挂单：
买价 = 99.1 - 0.1 = 99.0  ✅（刚好第二档）
卖价 = 99.2 + 0.1 = 99.3  ✅（刚好第二档）
```

**关键优势：**
- ✅ **精确**：直接基于订单簿第一档计算，不依赖中间价
- ✅ **动态**：订单簿变化时自动调整
- ✅ **简单**：固定偏移，易于理解和配置

### 3. 实时监控机制

系统持续监控两个指标：

**① 距离第一档的距离**
```
MM_LAYER_GUARD_NEAR_BPS=2  # 监控距离第一档2基点（约0.5美分）
```

**② 第一档深度**
```
MM_LAYER_GUARD_MIN_DEPTH_SHARES=5  # 第一档深度<5股时触发保护
```

### 4. 撤单重挂流程

当检测到订单变成或接近第一档时：

```
┌─────────────────────────────────────────┐
│  1. 检测到距离第一档 < 2基点              │
│                                        │
│  2. MM_SOFT_CANCEL_BPS=0.0003          │
│     → 立即触发软取消                      │
│                                        │
│  3. 撤单后重新计算第二档价格              │
│                                        │
│  4. MM_FAST_CANCEL_WINDOW_MS=500       │
│     → 500ms内重新挂单                    │
└─────────────────────────────────────────┘
```

```
4. 重新计算第二档：99.0 - 0.1 = 98.9
5. 500ms内重新挂单在 98.9

新状态：
买一：99.0（100股）
买二：98.9（50股）  ← 我们的买单重新挂在这里 ✅
```


## 积分规则要点

### Probable 积分计算

```
得分 = 基础分 × 距离权重 × 深度权重 × 时间权重
```

**关键因素：**
1. **距离权重**：距离市价越近，权重越高
   - 第二档：距离 ~0.1-0.2美分 ✅ 高权重
   - 第三档：距离 ~0.3-0.5美分 ❌ 低权重

2. **深度权重**：订单簿深度越大，权重越高
   - 第一档深度大：权重高 ✅
   - 第一档深度小：权重低 ❌

3. **时间权重**：挂单时间越长，累计得分越高
   - 长时间在第二档：持续得分 ✅
   - 频繁被吃单：中断得分 ❌

## 常见问题

### Q1: 为什么不直接挂在第一档？

**A:** 第一档风险太高：
- 立即被吃单的概率>50%
- 被吃后需要重新挂单，中断积分累计
- 频繁被吃会产生交易成本

第二档最佳：
- 被吃概率<5%
- 持续得分，积分效率更高
- 安全且有竞争力的得分

### Q2: 500ms响应时间够吗？

**A:** 500ms已经非常快：
- 人类反应时间：~200ms
- 网络延迟：~50-100ms
- 交易所处理：~50-100ms
- 500ms足够完成撤单重挂

实际测试：
- 大部分情况在200-300ms内完成
- 500ms是保守的上限

### Q3: 填单率1%是不是太低？

**A:** 1%是最佳平衡点：
- 太低（<0.5%）：说明挂单太保守，得分效率低
- 太高（>2%）：说明容易被吃，风险高

1%意味着：
- 每100个订单，1个被成交（正常做市）
- 99个被取消（快速调整位置）
- 既安全又能持续得分

### Q4: 如何判断策略是否有效？

**A:** 观察以下指标：

**短期（1小时）：**
- 订单位置应该在第二档占比>90%
- 撤单重挂频率：每分钟1-5次
- 填单率：~1%

**中期（1天）：**
- 积分得分持续上升
- 被吃单次数<总订单数的2%
- 净利润（扣除交易成本后）为正

**长期（1周）：**
- 积分排名稳定在前20%
- 总收益率稳定
- 系统稳定性高（无频繁暂停）
