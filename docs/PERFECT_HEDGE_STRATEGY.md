## 📚 核心概念

### 什么是完美对冲？

**完美对冲**是指在预测市场中持有**等量的 YES 和 NO**，使得无论价格如何变化，总价值保持不变。

### 数学原理

假设持有 **N 股 YES + N 股 NO**：

```
初始状态：YES = $0.60, NO = $0.40
总价值 = N × $0.60 + N × $0.40 = N × ($0.60 + $0.40) = N × $1.00

价格变化：YES → $0.80, NO → $0.20
总价值 = N × $0.80 + N × $0.20 = N × ($0.80 + $0.20) = N × $1.00 ✅

价格再变：YES → $0.30, NO → $0.70
总价值 = N × $0.30 + N × $0.70 = N × ($0.30 + $0.70) = N × $1.00 ✅
```

**关键条件**：YES + NO = 1（或 ≤ 1.05，允许小误差）

### 利润来源

**不是方向性收益**，而是**做市商价差**（bid-ask spread）：

1. 同时挂 YES 和 NO 的卖单（价格稍高于市场价）
2. 当一边被吃，立即市价卖出对边
3. 恢复 1:1 对冲状态
4. 赚取两边价差的利润

## 🎯 策略流程

### 初始化阶段

```
步骤1: 检查市场条件
  ├─ YES + NO ≤ 1.05? ✅ 继续
  └─ YES + NO > 1.05? ❌ 不适合此策略

步骤2: 建立初始对冲
  ├─ 买入 N 股 YES
  ├─ 买入 N 股 NO
  └─ 验证：N × YES价格 + N × NO价格 ≈ N
```

### 做市循环

```
状态: 持有 100 YES + 100 NO (完美对冲)

操作1: 双边挂单
  ├─ YES 卖单：$0.609 (市场价 $0.60 + 1.5%价差)
  └─ NO 卖单：$0.406 (市场价 $0.40 + 1.5%价差)

场景A: YES 卖单被吃 10 股
  ├─ 持仓变成：90 YES + 100 NO ❌ 不平衡
  ├─ 立即市价卖出 10 NO
  └─ 恢复：90 YES + 90 NO ✅ 平衡

场景B: NO 卖单被吃 15 股
  ├─ 持仓变成：90 YES + 75 NO ❌ 不平衡
  ├─ 立即市价卖出 15 YES
  └─ 恢复：75 YES + 75 NO ✅ 平衡
```

### 自动平衡机制

系统会实时监控对冲比例：

```
偏差 = |YES数量 - NO数量| / 平均数量

如果 偏差 > 5% (容忍度):
  ├ YES > NO? → 市价卖出多余 YES
  └ NO > YES? → 市价卖出多余 NO

恢复平衡后继续做市
```

## 📊 市场筛选

### 适合的市场

✅ **YES + NO ≤ 1.05**（允许 5% 误差）

示例：
- YES=$0.60, NO=$0.40 → 和=$1.00 ✅ 极好
- YES=$0.55, NO=$0.44 → 和=$0.99 ✅ 良好
- YES=$0.50, NO=$0.50 → 和=$1.00 ✅ 完美

### 不适合的市场

❌ **YES + NO > 1.05**

示例：
- YES=$0.70, NO=$0.50 → 和=$1.20 ❌ 不适合
- YES=$0.80, NO=$0.40 → 和=$1.20 ❌ 不适合

### 风险警告

**如果 YES+NO 从 1.00 变成 1.20**：
```
原来：100 YES + 100 NO = 100 × $1.00 = $100
现在：100 YES + 100 NO = 100 × $1.20 = $120 ❌ 对冲失效！

风险：价格可能回归到 1.00，导致亏损
```

**应对措施**：
1. 实时监控 YES+NO 的和
2. 超过 1.05 时立即平仓
3. 等待回归到 1.05 以下再入场

## 📈 性能预期

### 收益来源

**纯价差收益**（与价格方向无关）：

```
每次成交收益 = 订单大小 × 价差 × 2

示例：
订单大小 = $25
价差 = 1.5%
每次收益 = $25 × 0.015 × 2 = $0.75

每天成交 20 次：
日收益 = $0.75 × 20 = $15
月收益 = $15 × 30 = $450
```

### 风险控制

**最大风险**：YES+NO 从 1.00 涨到 1.20 后又回归

```
最坏情况：
- YES 从 $0.50 涨到 $0.70
- NO 从 $0.50 涨到 $0.50
- 和从 $1.00 变成 $1.20

如果持有 100 YES + 100 NO：
原来价值 = 100 × $1.00 = $100
峰值价值 = 100 × $1.20 = $120
回归价值 = 100 × $1.00 = $100

亏损 = $0 (如果能及时平仓)
实际亏损 = 滑点成本 (~2-3%)
```

## ⚠️ 注意事项

### 必须监控

1. ✅ **YES+NO 的和**：超过 1.05 立即平仓
2. ✅ **流动性深度**：确保能快速成交
3. ✅ **滑点成本**：平衡时的实际滑点
4. ✅ **手续费**：频繁交易的成本

### 常见错误

❌ **错误 1**: 在 YES+NO>1.05 的市场使用
- 后果：对冲失效，可能亏损 10-20%

❌ **错误 2**: 容忍度设置过大（>10%）
- 后果：对冲不精确，风险增加

❌ **错误 3**: 忽略流动性
- 后果：无法及时平衡，扩大滑点

❌ **错误 4**: 关闭自动平衡
- 后果：需要手动平衡，容易出错

### 最佳实践

✅ **只在高流动性市场使用**
- 24小时交易量 > $10,000
- 订单簿深度 > $1,000

✅ **实时监控 YES+NO**
- 每分钟检查一次
- 超过 1.05 立即平仓

✅ **设置合理容忍度**
- 新手：3%（更严格）
- 经验：5%（平衡）
- 激进：10%（需要频繁平衡）

✅ **控制最大规模**
- 新手：100-200 股
- 经验：200-500 股
- 激进：500-1000 股
